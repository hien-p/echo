2026-05-08 feat

v0.2 privacy: gating, anonymous nullifier, multi-admin caps

Three privacy-tier features land in one push. Gating: checkGating() enforces token / NFT / SuiNS predicates client-side; FormViewer banner with a "Verify again" button. Anonymous nullifier: submit_anonymous now records each commitment in a Table on the Form and aborts ECommitmentAlreadyUsed on double-submit. Browser derives a deterministic commitment via deriveCommitment(walletAddress, formId) — wallet signs a canonical personal message; SHA-256 over the signature gives the 32-byte nullifier. Same wallet/form → same hash; chain catches the second submit without seeing the address. Multi-admin OR-of-N: create_form takes an extra_admins: vector<address> param and mints one FormOwnerCap per address. FormBuilder picks up a Co-admins textarea (parses comma/newline lists, shows "OR-of-N — any one can decrypt"). 18 Move tests + 21 dapp tests pass.

2026-05-07 chore

RAG pipeline verified end-to-end on AdminOnly form

Seeded a fresh AdminOnly form with realistic per-field answers (salary numbers, burnout-style concerns, churn-risk choices), indexed via /api/insights/index_form against a Memwal namespace, queried via /api/insights/query through OpenRouter. 2026-05-07 refactor

FormAdmin loads ~40% faster + node runtime fix

/forms/[id]/admin previously chained formQuery → submissionsQuery sequentially and re-fetched every 200 SubmissionMade events globally just to client-filter to the form. Split into parallel submissionEventsQuery (server-side filtered by form_id via All+MoveEventField, falls back gracefully) and a downstream payload-enrichment stage that only runs for Public tier. Added staleTime across the three queries so navigation back doesn't trigger fresh reads. Decrypt routes switched to nodejs runtime (Edge sandbox lacked AbortSignal.any needed by @mysten/seal) and stopped using WalrusClient.readBlob in favor of HTTP aggregator fetches (~1s vs 60s).

2026-05-07 feat

Browser-driven Insights indexer (zero-trust path)

New /api/insights/index_one endpoint accepts only { formId, text } — plaintext that the browser already decrypted. Server never sees ciphertext, never holds Seal keys, no decrypt capability. /forms/[id]/admin picks up an "Index for Insights" button (visible to non-demo cap holders for non-Public forms) that creates one wallet-signed SessionKey, decrypts every submission locally, and posts only flattened answer text. The server-side /api/insights/index_form path stays available for demo-owned forms; this new path is the privacy-preserving option for real users.

2026-05-07 feat

Insights indexer covers all encrypted tiers

/api/insights/index_form now indexes AdminOnly, Threshold, and Conditional forms whose owner matches the indexer keypair (typically DEMO_ADMIN_SECRET_KEY). Server looks up the FormOwnerCap on its own derived address, builds the matching seal_approve_* PTB, signs the SessionKey, decrypts each submission, and remembers the flattened text in Memwal. /insights dropdown stops greying out non-Public forms whenever the demo toggle is on. Indexer key priority: DEMO_ADMIN_SECRET_KEY > MEMWAL_PRIVATE_KEY.
