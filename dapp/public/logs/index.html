2026-05-07 feat

Browser-driven Insights indexer (zero-trust path)

New /api/insights/index_one endpoint accepts only { formId, text } — plaintext that the browser already decrypted. Server never sees ciphertext, never holds Seal keys, no decrypt capability. /forms/[id]/admin picks up an "Index for Insights" button (visible to non-demo cap holders for non-Public forms) that creates one wallet-signed SessionKey, decrypts every submission locally, and posts only flattened answer text. The server-side /api/insights/index_form path stays available for demo-owned forms; this new path is the privacy-preserving option for real users.

2026-05-07 feat

Insights indexer covers all encrypted tiers

/api/insights/index_form now indexes AdminOnly, Threshold, and Conditional forms whose owner matches the indexer keypair (typically DEMO_ADMIN_SECRET_KEY). Server looks up the FormOwnerCap on its own derived address, builds the matching seal_approve_* PTB, signs the SessionKey, decrypts each submission, and remembers the flattened text in Memwal. /insights dropdown stops greying out non-Public forms whenever the demo toggle is on. Indexer key priority: DEMO_ADMIN_SECRET_KEY > MEMWAL_PRIVATE_KEY.

2026-05-07 feat

Encrypted demo forms + seed-submissions script

createSampleForm.ts picks up three new variants — FORM=admin, FORM=threshold, FORM=timelocked — that mint AdminOnly, Threshold(1/1), and TimeLocked(now+5min) forms so the demo decrypt path actually has something to decrypt. New seedSubmissions.ts reads a form's tier from chain, generates plausible answers from the schema, encrypts via Seal when the tier requires it, and submits N=3 fake responses through the publish admin keypair.

2026-05-07 feat

Demo admin: animated switch + cross-page demo views

DemoAdminToggle now renders an iOS-style switch with a sliding thumb (300ms ease-out). Connect Wallet hides while demo mode is on so visitors can browse anonymously. /forms, /insights, and /reputation all key off the demo address when the toggle is on — the demo wallet's caps + badge are visible without connecting. Mint/Claim stay disabled in demo mode (those need a real wallet).

2026-05-07 feat

Demo admin toggle + server-side Seal decrypt

New header pill flips on a demo-mode admin path: /api/demo/admin/{list,decrypt} route handlers sign Seal SessionKey server-side using DEMO_ADMIN_SECRET_KEY and decrypt submissions for any form whose owner matches NEXT_PUBLIC_DEMO_ADMIN_ADDRESS. Lets visitors explore admin features without holding the FormOwnerCap. publish/transferDemoCaps.ts re-homes existing caps to the demo address in one PTB. Demo-only — real AdminOnly tiers stay wallet-driven.

2026-05-07 feat

Zero-gas form authoring + JSON edit mode

/api/walrus/upload proxy now forwards bytes to the Walrus testnet publisher — schema/metadata/payload uploads pay zero gas. Combined with sponsored create_form and sponsored submit, both authors and respondents can ship without holding SUI. Builder picks up a Visual ↔ Edit-JSON toggle: power users edit raw { metadata, tier, schema } in a textarea, Apply validates JSON before writing back to state.
